API Guide
Product Catalog
Occasional
Retrieve the product catalog and display on the product selection screen. Since the changes in product catalogue are occasional, it is recommended to cache the products in-memory for fast retrieval.
GET Product Catalogue
Create Account
Frequent
After collecting user details, it can be used to create an account. Optionally, an account can be created at this step using the API mentioned below.
This API allows to create Account, BillTo and SoldTo contact in one invocation
Optionally, a Billing Account can be created during Subscribe (last step).
POST New Account
Create Contact
Occasional
If there is a requiement to create Contacts and Account separately, then create Contact using below API and then create Account. Pass the contact id (received in create contact response) to create Account API.
This API allows to create contact which can be used to set on Account's BillTo or SoldTo or both contacts.
Optionally, a Contact can be created on Account creation or during Subscribe (last step).
POST New Contact
Create Payment Method
Frequent
Collect the payment method details and consume API to add a payment method to a Billing Account.
Optionally, a Payment Method can be created during Subscribe (this step). OR you can use Zuora HPM pages to create a Payment Method and use the PaymentMethodId to set on Account.
POST New Payment Method
Apply Promo Code
Promotional
Zuora Promo Code app allows to manage the promotional campaigns. Using the APIs promotional discounts can be applied during sign up process.
Promo Code APIs Swaggger
GET /api/v1/tools/promos/{code}/validate
PUT /api/v1/tools/promos/{code}/consume
Create Subscription
Frequent
After collecting following information build a payload to create a subscription in Zuora.
  • - Billing Account
  • - Contact
  • - Payment Method
  • - Selected Plan
Optionally, on subscribe API callout, an invoice can be generated and payment can also be collected using the added electronic payment method.

Create Payment Method
Independent
Payment Method can independently created either before or after Billing Account creation or during subscription creation. If the payment method is created before, then use the payment method id attach to Billing Account.
Zuora administrator will need to configure the payment gateway. Optionally, HPM pages for capturing payment method using hosted pages.
Promotion Code
On Signup
App administrator needs to configure campaigns and promocodes to be able to apply during sign up flow. Zuora Promo Code app purchased separately as per requirements. Please contact Zuora sales team.
  • Step 1: Invoke the desired validate API as per the requirements
  • Step 2: If validation success, invoke the desired consume API to retrieve the promotional discount rate plan configured in Promo Code App.
  • Step 3: The consume API response contains discount rate plan identifiers which will be used to add promotional discount to subscription.
Create Subscription
On Signup
Use the information collected for a Billing Account, Contacts, a Payment Method, a Promotion, and product details to create a subscription.
On successful subscription creation, the response will contain the Subscription number, Account number.
Optionally, if the invoice generation and payment collection flags are true, then, an invoice will be generated and payment will also be collected against the payment method.
